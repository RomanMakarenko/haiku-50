# Haiku 50

## Overview

Веб-застосунок для генерації хайку через OpenAI API. Користувач вводить 3-7 ключових слів, обирає мову та натискає кнопку генерації. Хайку — короткий японський трирядковий вірш: момент, образ, настрій. Суворий лічильник складів 5-7-5 не потрібен.

### Унікальні фічі
- **50 Васабі (Spiciness Level):** кнопка з іконкою васабі піднімає рівень гостроти від 0 до 6 — від спокійної класики до гострого, майже абсурдного тексту. Рівень гостроти керує грайливістю, абсурдом і непередбачуваністю тексту
- **Історія генерацій:** останні 100 хайку зберігаються локально в браузері (localStorage). Серверне in-memory-сховище — лише додаткова опція, не основна вимога
- **Дизайн:** Bento Grid, максимально мінімалістичний, стилізований під японську естетику — каліграфічний пензель для кнопки, сувій, що розкривається, бонсай на тлі
- **Безпека контенту:** застосунок не допускає мат, агресію або заборонений контент — ні в ключових словах, ні в згенерованій хайку

## Architecture

### Client → Server → OpenAI

1. **Frontend** (`public/`) надсилає POST-запит на `/api/generate` з тілом:
   ```json
   {
     "keywords": "сакура, дощ, тиша",
     "language": "uk",
     "spiciness": 3
   }
   ```
2. **Express-сервер** отримує запит, підставляє `OPENAI_API_KEY` із змінної середовища, формує промпт і викликає OpenAI API.
3. Якщо OpenAI відповідає — сервер повертає `{ "haiku": "...", "fallback": false }`.
4. Якщо OpenAI не відповідає — сервер повертає fallback-хайку відповідною мовою з `{ "haiku": "...", "fallback": true }` (код 200).
5. **Frontend** відображає хайку в анімованому сувої. Якщо `fallback === false` — зберігає в localStorage; якщо `fallback === true` — не зберігає.

### 50 Васабі (Spiciness Level)
- Кнопка з **іконкою васабі** — при кожному кліку збільшує лічильник, після 6 скидає на 0
- Рівень 0 — спокійна класика
- Рівень 6 — максимально гостро, майже абсурдно
- Рівень гостроти керує **грайливістю, абсурдом і непередбачуваністю** тексту
- Біля лічильника — знак питання; наведення курсора пояснює логіку (tooltip)
- Значення передається в API як параметр `spiciness`

### Історія
- Зберігається в `localStorage` під ключем `haikuHistory`
- Максимум 100 записів
- Кожен запис: `{ haiku, language, spiciness, timestamp, keywords }`
- Видаляється найстаріший запис при перевищенні ліміту
- Серверне in-memory-сховище — лише додаткова опція, не основна вимога

### Поле ключових слів
- Поле введення **НЕ очищається** після генерації хайку — введені слова залишаються видимими
- Поруч із полем — окрема кнопка **очищення** (іконка ✕ аналогічно)
- Це дозволяє легко генерувати нові варіації на ту саму тему

### Валідація

**Client-side (перед відправкою запиту):**
| Умова | Повідомлення | Кнопка генерації |
|---|---|---|
| Ключових слів < 3 | "Введіть щонайменше 3 ключові слова або фрази" | Заблокована |
| Ключових слів > 7 | "Не більше 7 ключових слів або фраз" | Заблокована |
| Мову не обрано | "Оберіть мову генерації" | Заблокована |
| Усі умови виконано | — | Активна |

**Server-side (після отримання запиту):**
- Повторна валідація keywords (3-7), language, spiciness (0-6)
- При помилці: 400 + `{ "error": "Опис українською" }`

**Error handling на клієнті:**
| Ситуація | Що бачить користувач |
|---|---|
| Сервер недоступний (network error) | "Сервер тимчасово недоступний. Спробуйте пізніше." |
| Таймаут запиту (> 10 с) | "Запит зайняв занадто багато часу. Спробуйте ще раз." |
| Будь-яка інша помилка від сервера | Повідомлення з поля `error` у відповіді |
| Fallback-хайку | Відображається в сувої, але не зберігається в історію |

Усі помилки відображаються у вигляді **inline-повідомлення** під формою введення (червоний текст або alert-блок).

### Content Safety
- **Ключові слова:** перевіряються на наявність мату, агресії, забороненого контенту. При виявленні — помилка з повідомленням: "Видаліть нецензурні або агресивні слова"
- **OpenAI Prompt:** системний промпт містить інструкцію: "Не використовуй мат, агресію, образи або заборонений контент. Якщо ключові слова провокаційні — створи безпечний, поетичний текст на основі їхнього образу"
- **Відповідь OpenAI:** якщо модель повернула неприйнятний контент — сервер повертає помилку, і хайку не зберігається в історію

### Fallback при недоступності OpenAI API

Якщо OpenAI API не відповідає (таймаут, NetworkError, некоректна відповідь), сервер повертає **заздалегідь написану хайку мовою, яку обрав користувач**, замість помилки 502:

| en (English) | Servers hum in vain,<br>Silence where the words should be —<br>Sadly we apologize. |
| de (Deutsch) | Server rauschen leer,<br>Stille statt der Poesie —<br>Bitte hab Geduld. |
| ja (日本語) | サーバーが沈み、<br>言葉の代わりに静けさ —<br>申し訳ありません。 |
| fr (Français) | Les serveurs sont las,<br>Le silence fait son œuvre —<br>Nous sommes désolés. |
| es (Español) | Los servidores<br>en silencio responden —<br>Lo sentimos. |
| it (Italiano) | I server tacciono,<br>nessuna poesia arriva —<br>Ci dispiace molto. |
| pt (Português) | Servidores mudos,<br>O silêncio é a resposta —<br>Nos desculpemos. |
| pl (Polski) | Serwery śpią,<br>Cisza zamiast wiersza —<br>Przepraszamy. |
| zh (中文) | 服务器沉默，<br>寂静代替了诗句 —<br>我们很抱歉。 |

**Логіка:**
- Якщо OpenAI API повертає помилку або таймаут — сервер повертає `200 OK` з fallback-хайку відповідною мовою
- Якщо сервер взагалі недоступний (клієнт не може достукатись до Render) — клієнт показує стандартне повідомлення "Сервер тимчасово недоступний"
- Fallback-хайку **не зберігається** в історію (історія тільки для справжніх генерацій)
- На фронт відповідь приходить з додатковим полем: `{ "haiku": "...", "fallback": true }`

**Client-side (обов'язково):**
| Тест | Що перевіряє |
|---|---|
| Введено < 3 слів → кнопка заблокована | Валідація до відправки |

## API Design

### POST /api/generate
**Request body:**
```json
{
  "keywords": "string (3-7 слів/фраз через кому)",
  "language": "uk | en | de | ja | fr | es | it | pt | pl | zh | ko | ar",
  "spiciness": "number (0-6)"
}
```

**Response (200) — успішна генерація:**
```json
{
  "haiku": "рядок 1\nрядок 2\nрядок 3",
  "fallback": false
}
```

**Response (200) — fallback (OpenAI недоступний):**
```json
{
  "haiku": "рядок 1\nрядок 2\nрядок 3",
  "fallback": true
}
```

**Response (400) — валідація:**
```json
{
  "error": "Введіть щонайменше 3 ключові слова або фрази"
}
```
```json
{
  "error": "Не більше 7 ключових слів або фраз"
}
```
```json
{
  "error": "Оберіть мову генерації"
}
```
```json
{
  "error": "Видаліть нецензурні або агресивні слова"
}
```

**Response (500) — внутрішня помилка сервера:**
```json
{
  "error": "Щось пішло не за планом. Спробуйте пізніше."
}
```

### OpenAI Prompt
Системний промпт формується динамічно:
```
Ти — японський поет. Напиши хайку мовою {language} на основі слів: {keywords}.
Рівень гостроти (spiciness): {0-6}.
0 = класичний спокійний вірш, 6 = максимально гострий — грайливий, абсурдний, непередбачуваний текст.
Хайку має бути з трьох рядків. Суворий 5-7-5 не потрібен.
Не використовуй мат, агресію, образи або заборонений контент.
Якщо ключові слова провокаційні — створи безпечний, поетичний текст на основі їхнього образу.
Поверни ТІЛЬКИ три рядки хайку, без додаткового тексту.
```

## Setup

### Development
1. Клонувати репозиторій
2. Створити `.env` у корені:
   ```
   OPENAI_API_KEY=sk-...
   PORT=3000
   ```
3. `npm install`
4. `node api/server.js` (або `npm run dev` з nodemon)

### Production (Render)
1. GitHub репозиторій підключити до Render
2. **Type:** Web Service
3. **Build Command:** `npm install`
4. **Start Command:** `node api/server.js`
5. **Environment Variables:** додати `OPENAI_API_KEY`
6. **Health Check Path:** `/api/health`

## Security

- **API key НІКОЛИ не потрапляє в клієнтський код і GitHub**
- **Обов'язково:** API key зберігається у файлі `.env` в корені проєкту для локальної розробки
  - `.env` додано в `.gitignore` — ніколи не комітиться
  - Приклад структури: `.env.example` з плейсхолдером комітиться в репозиторій
- На Render: ключ вводиться через Dashboard → Environment Variables (не через файл)
- На GitHub: у `.env.example` зберігається тільки заглушка, `.env` в `.gitignore`
- Рекомендується зробити репозиторій публічним (Render безкоштовно працює з публічними), але ключ залишається прихованим

## Notes

- Модель **строго одна**: `gpt-5-nano-2025-08-07`. Не міняти.
- Express сервер слухає порт із `process.env.PORT || 3000`
- Статика віддається через `express.static('public')`
- CORS не потрібен — фронт і бек на одному домені
- У Render сервер може "засинати" після 15 хв бездіяльності; перший запит після простою займає ~30 с
- Після генерації поле ключових слів зберігає введений текст; очищення — окремою кнопкою
- Історія зберігається виключно в localStorage (100 записів); in-memory на сервері — опціонально
- Content safety реалізується на трьох рівнях: валідація ключових слів → інструкція в промпті → перевірка відповіді
- UI виконується в стилі Bento Grid — мінімалістичний, асиметрична сітка блоків
- Набір мов — рівно 12; нові мови додаються лише через оновлення списку в коді та промпті
